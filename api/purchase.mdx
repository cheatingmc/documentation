# Purchase Endpoint

Purchase Minecraft accounts programmatically using your credit balance or cryptocurrency.

## Endpoint

```
POST https://api.nicealts.com/purchase
```

## Request Body

<ParamField path="account_type" type="string" required>
  Type of account to purchase:
  - `unbanned_accesstoken`
  - `unbanned_cookie`
  - `unbanned_emailpass`
  - `banned_accesstoken`
  - `banned_cookie`
  - `banned_emailpass`
</ParamField>

<ParamField path="quantity" type="integer" required>
  Number of accounts to purchase (1-50 per request)
</ParamField>

<ParamField path="payment_method" type="string" default="credits">
  Payment method: `credits` or `crypto`
</ParamField>

## Request Examples

<CodeGroup>

```bash cURL
curl -X POST "https://api.nicealts.com/purchase" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "account_type": "unbanned_accesstoken",
    "quantity": 5,
    "payment_method": "credits"
  }'
```

```python Python
import requests

headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

data = {
    "account_type": "unbanned_accesstoken",
    "quantity": 5,
    "payment_method": "credits"
}

response = requests.post(
    "https://api.nicealts.com/purchase",
    headers=headers,
    json=data
)

result = response.json()
print(result)
```


## Response Fields

<ResponseField name="success" type="boolean">
  Whether the purchase request was successful
</ResponseField>

<ResponseField name="purchase_id" type="string">
  Unique identifier for this purchase
</ResponseField>

<ResponseField name="accounts" type="array">
  Array of purchased accounts (only for credit purchases)
  
  <Expandable title="Account Object">
    <ResponseField name="username" type="string">
      Minecraft username
    </ResponseField>
    
    <ResponseField name="access_token" type="string">
      Access token for authentication (AccessToken accounts)
    </ResponseField>
    
    <ResponseField name="cookie" type="string">
      Session cookie (Cookie accounts)
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Email address (Email:Pass accounts)
    </ResponseField>
    
    <ResponseField name="password" type="string">
      Password (Email:Pass accounts)
    </ResponseField>
    
    <ResponseField name="uuid" type="string">
      Minecraft UUID
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="payment_details" type="object">
  Payment information (only for crypto purchases)
</ResponseField>

## Example Integration

Complete purchase flow with error handling:

```python
import requests
import time

def purchase_accounts(api_key, account_type, quantity):
    headers = {
        "Authorization": f"Bearer {api_key}",
        "Content-Type": "application/json"
    }
    
    # Check stock first
    stock_response = requests.get(
        "https://api.nicealts.com/stock",
        headers=headers
    )
    
    if stock_response.status_code != 200:
        return {"error": "Failed to check stock"}
    
    stock = stock_response.json()
    available = stock['stock'][account_type]['available']
    
    if available < quantity:
        return {"error": f"Insufficient stock. Available: {available}"}
    
    # Make purchase
    purchase_data = {
        "account_type": account_type,
        "quantity": quantity,
        "payment_method": "credits"
    }
    
    response = requests.post(
        "https://api.nicealts.com/purchase",
        headers=headers,
        json=purchase_data
    )
    
    if response.status_code == 200:
        result = response.json()
        print(f"âœ“ Purchase successful! Received {len(result['accounts'])} accounts")
        return result
    else:
        return {"error": response.json()}

# Example usage
result = purchase_accounts(
    "your_api_key",
    "unbanned_accesstoken",
    5
)
```

## Error Responses

<ResponseField name="400 Bad Request" type="error">
  Invalid request parameters
  ```json
  {
    "error": "Invalid account type",
    "status": 400
  }
  ```
</ResponseField>

<ResponseField name="402 Payment Required" type="error">
  Insufficient credits
  ```json
  {
    "error": "Insufficient credits",
    "status": 402,
    "required": 50,
    "available": 25
  }
  ```
</ResponseField>

<ResponseField name="409 Conflict" type="error">
  Insufficient stock
  ```json
  {
    "error": "Insufficient stock",
    "status": 409,
    "requested": 10,
    "available": 3
  }
  ```
</ResponseField>

<ResponseField name="429 Too Many Requests" type="error">
  Rate limit exceeded
  ```json
  {
    "error": "Rate limit exceeded",
    "status": 429,
    "retry_after": 60
  }
  ```
</ResponseField>

## Rate Limits

- Maximum 50 accounts per purchase request
- Maximum 10 requests per minute
- Daily purchase limit based on account tier

<Tip>
  For bulk purchases exceeding 50 accounts, make multiple requests with appropriate delays between them.
</Tip>
